# Getting Started with FlowStory

FlowStory is an AI-agent-first framework for creating animated, step-by-step architecture flow diagrams. You define your diagram in JSON, and the engine handles canvas rendering, animation, and interactive playback.

## Creating a Diagram with AI Tools

FlowStory is designed to be generated by AI agents, not manually coded. There are two approaches:

### Option A: Vanilla Claude Code (Recommended for Iterative Design)

This is how the original AI Gateway visualizer was built — no plugins needed.

**Setup**: Just use Claude Code as-is.

**Workflow**:

1. Describe what you want:
   > "Create a flowstory diagram showing a 3-tier web app: client → load balancer → API server → database. Show request and response flows."

2. Claude generates `diagram.json`

3. Preview locally:
   ```bash
   cd your-diagram/
   npx flowstory dev
   # Open http://localhost:9000
   ```

4. **Screenshot feedback loop** — this is the key technique:
   - Open the diagram in your browser
   - Take a screenshot of what you see
   - Paste the screenshot into Claude Code
   - Describe what to change: "move the database box down 50px", "arrow 5 overlaps arrow 3, shift it right", "the EPP box is too close to IPP"
   - Claude updates the JSON, you refresh and repeat

5. Keep iterating until it looks right

**Why this works**: Claude can see your screenshots and adjust exact pixel coordinates, colors, and layout. The back-and-forth converges fast — usually 5-10 iterations for a polished diagram.

### Option B: Claude Code + Frontend Design Plugin (Better Initial Aesthetics)

Install the optional frontend-design plugin for higher-quality initial UI generation:

```
/plugin marketplace add anthropics/claude-code
/plugin install frontend-design@claude-plugins-official
```

**What it adds**: When Claude generates HTML/CSS (for custom template styling, landing pages around your diagram, etc.), the plugin produces more distinctive, production-grade aesthetics out of the box. It does NOT change how diagram JSON is generated — that's schema-driven either way.

**When to use**: If you're customizing the template HTML, building a gallery page, or creating a landing page around your diagram. For the diagram JSON itself, both options are equivalent.

**The screenshot workflow still applies** — the plugin improves starting quality, but you'll still iterate via screenshots for pixel-perfect results.

### Option C: Other AI Tools (GPT, Gemini, etc.)

Any AI tool can generate FlowStory diagrams. Copy the prompt from `docs/agent-prompt.md` (or run `npx flowstory prompt`) and paste it into your tool of choice. It contains the full JSON schema and examples.

The screenshot feedback loop is specific to Claude Code (which accepts image input in the CLI). With other tools, you may need to describe layout issues in words instead of screenshots.

## Quick Start

### 1. Create a project

```bash
mkdir my-diagram && cd my-diagram
npm init -y
npm install flowstory
```

### 2. Create diagram.json

Start with the hello-world example or use an AI tool to generate one:

```json
{
  "meta": { "title": "My Flow Diagram" },
  "canvas": { "width": 800, "height": 400 },
  "nodes": {
    "client": { "x": 50, "y": 150, "w": 100, "h": 50, "label": "Client", "type": "icon" },
    "server": { "x": 300, "y": 150, "w": 200, "h": 80, "label": "API Server", "color": "#58a6ff" },
    "db":     { "x": 650, "y": 150, "w": 150, "h": 60, "label": "Database", "color": "#3fb950" }
  },
  "flows": {
    "main": {
      "label": "Request Flow",
      "steps": [
        { "text": "Client sends request", "mode": "arrow", "from": "client", "to": "server", "color": "#58a6ff", "num": 1 },
        { "text": "Server queries DB", "mode": "arrow", "from": "server", "to": "db", "color": "#58a6ff", "num": 2 },
        { "text": "DB responds", "mode": "arrow", "from": "db", "to": "server", "color": "#3fb950", "num": 3 },
        { "text": "Server responds", "mode": "arrow", "from": "server", "to": "client", "color": "#3fb950", "num": 4 }
      ]
    }
  }
}
```

### 3. Create index.html

```html
<!DOCTYPE html>
<html>
<head>
  <title>My Flow Diagram</title>
  <link rel="stylesheet" href="node_modules/flowstory/templates/style.css">
</head>
<body>
  <canvas id="canvas"></canvas>
  <script type="module">
    import { FlowStory } from 'flowstory';
    const viz = new FlowStory(document.getElementById('canvas'));
    await viz.load('./diagram.json');
  </script>
</body>
</html>
```

### 4. Preview

```bash
npx flowstory dev
# Open http://localhost:9000
```

### 5. Deploy

```bash
npx flowstory build --inline
# Produces dist/index.html — deploy to GitHub Pages or any static host
```

## Tips for AI-Assisted Diagram Creation

1. **Start with structure, refine layout later**: First get all nodes and flows correct, then adjust positions via screenshots.

2. **Use consistent color conventions**:
   - Blue (`#58a6ff`) for request path
   - Green (`#3fb950`) for response path
   - Orange (`#f0883e`) for auth/error flows
   - Purple (`#d2a8ff`) for plugin containers

3. **Coordinate system**: Origin is top-left. Default canvas is 1250x1050. Negative x values place nodes outside the main boundary (useful for external actors).

4. **Arrow tuning**: When arrows overlap, adjust `edge.fromXOffset` and `edge.toXOffset` to spread them apart. Use `edge.fromSide` / `edge.toSide` to control which side of a node the arrow connects to.

5. **The screenshot loop is your friend**: Don't try to get coordinates perfect on the first try. Generate, screenshot, adjust, repeat.
